# 训练总流程（新手详细版）

这份文档给第一次接触本仓库的同学使用，目标是：

- 先把环境配对
- 再把流程跑通
- 最后能读懂结果和分析 notebook

## 0. 仓库结构先看懂

本仓库现在有四条并列教学线：

- ANI：`minimal_adl_ethene_butadiene`
- KREG：`minimal_adl_ethene_butadiene_KREG`
- MACE：`minimal_adl_ethene_butadiene_mace`
- NequIP（预留）：`minimal_adl_ethene_butadiene_nequip`

`docs/` 目录对应为：

- `docs/ani/`
- `docs/kreg/`
- `docs/mace/`
- `docs/nequip/`

## 1. 环境和工程一一对应

- ANI：`ADL_env`
- KREG：`ADL_KREG`
- MACE：`ADL_MACE`
- NequIP：`ADL_NequIP`

不要混用环境，否则最常见的问题就是脚本能 import、但训练或标注阶段直接失败。

## 2. 先跑门禁检查

进入对应工程目录后，固定先执行：

```bash
python scripts/check_environment.py --config configs/base.yaml --strict
```

如果你用的是 ANI、KREG 或 MACE 这三条 TS + MD 主线，并且已经准备好 TS seed，再额外执行：

```bash
python scripts/prepare_ts_seed.py --config configs/base.yaml
python scripts/check_environment.py --config configs/base.yaml --strict --test-mlatom-xtb
```

## 3. 各模型当前推荐入口

### 3.1 ANI

看：`docs/ani/训练流程.md`

ANI 当前默认是：

`TS seed -> Round 0 标注 -> 训练 -> 双向 MD -> 首个不确定点返回 -> 再标注`

### 3.2 KREG

看：`docs/kreg/训练流程.md`

KREG 当前也已经切到：

`TS seed -> Round 0 标注 -> 训练 -> 双向 MD -> 首个不确定点返回 -> 再标注`

### 3.3 MACE

看：`docs/mace/训练流程.md`

MACE 现在与 ANI 逻辑对齐，只保留模型后端不同：

`TS seed -> Round 0 标注 -> MACE 训练 -> 双向 MD -> 首个不确定点返回 -> 再标注`

默认资源是：

- `default` 队列
- `16` 核
- `360:00:00`
- 默认 `cpu`

### 3.4 NequIP

当前仍是预留目录，优先参考现有文档，不建议作为首选教学线路。

## 4. MACE 最小联通测试

```bash
cd minimal_adl_ethene_butadiene_mace
python scripts/prepare_ts_seed.py --config configs/base.yaml
python scripts/sample_ts_initial_conditions.py --config configs/base.yaml --round-index 0 --num-initial-conditions 10
python scripts/run_xtb_labels.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json --submit-mode local
python scripts/run_target_labels.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json --submit-mode local
python scripts/update_cumulative_manifest.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json
python scripts/build_delta_dataset.py --config configs/base.yaml --manifest data/processed/cumulative_labeled_manifest.json
python scripts/train_main_model.py --config configs/base.yaml --submit-mode local --device cpu
python scripts/train_aux_model.py --config configs/base.yaml --submit-mode local --device cpu
python scripts/run_md_sampling.py --config configs/base.yaml --round-index 1 --num-initial-conditions 4 --maximum-propagation-time 10 --time-step 0.5 --save-interval-steps 2 --submit-mode local
python scripts/select_md_frames.py --config configs/base.yaml --round-index 1 --max-new-points 5
```

## 5. MACE 完整第一轮

```bash
cd minimal_adl_ethene_butadiene_mace
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --submit-mode-labels pbs \
  --submit-mode-train pbs \

```

## 6. 跑完后去哪里看

每个工程都只看自己的目录，不要串目录：

- `models/`：模型与训练产物
- `results/`：流程结果、选点结果、轨迹摘要
- `data/processed/`：累计 manifest 与数据集

对 MACE 来说，最关键的新文件是：

- `results/ts_seed_summary.json`
- `results/round_001_md_frame_manifest.json`
- `results/round_001_selection_summary.json`
- `data/processed/cumulative_labeled_manifest.json`
- `models/training_state.json`

## 7. notebook 怎么用

每条线都各有 notebook：

- `docs/ani/数据分析.ipynb`
- `docs/kreg/数据分析.ipynb`
- `docs/mace/数据分析.ipynb`
- `docs/nequip/数据分析.ipynb`

建议先把对应工程的产物跑出来，再打开 notebook。

## 8. 新手最容易踩的坑

- 混用 conda 环境
- 没先跑 `check_environment --strict`
- 还在用旧的“静态 400 点几何池”理解当前主线
- 以为 MD 要先跑完整条轨迹再统一筛选；现在是“遇到首个不确定点就停”
- 跑的是 A 工程，却去 B 工程目录看结果
