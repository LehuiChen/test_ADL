# 训练总流程（新手详细版）

这份文档只保留当前有效的三条主线：ANI、MACE、KREG。三条线除了机器学习后端和训练环境不同，其余流程已经对齐。

## 1. 先认目录

- ANI：`minimal_adl_ethene_butadiene`
- MACE：`minimal_adl_ethene_butadiene_mace`
- KREG：`minimal_adl_ethene_butadiene_KREG`

对应文档目录：

- `docs/ani/`
- `docs/mace/`
- `docs/kreg/`

NequIP 已经从仓库中移除，不再作为可选主线。

## 2. 三条线的共同流程

现在三条线都按下面这条主线运行：

`TS.log -> prepare_ts_seed -> sample_ts_initial_conditions -> labels -> build_delta_dataset -> train -> bidirectional MD -> stop_on_first_uncertain_point -> select_md_frames -> relabel`

关键理解：

- round 0 会先标注一批 TS 附近初始条件，训练出第一版势
- 后续 MD 用当前 ML 势传播轨迹
- 每条轨迹只在“第一个不确定点”停下并返回候选样本
- 不是把整条轨迹上的所有点都拿去做 Gaussian 标注

## 3. 环境一一对应

- ANI 训练环境：`ADL_env`
- MACE 训练环境：`ADL_MACE`
- KREG 训练环境：`ADL_KREG`

当前默认资源策略：

- ANI：baseline / target 标签走 `default` 队列、`16` 核；训练和 MD 走 `GPU` 队列，`Req'dTime = 360:00:00`
- MACE：所有阶段走 `N4` 队列、`24` 核
- KREG：baseline / target 走 `N4` 队列、`24` 核；training / uncertainty / md_sampling 走 `GPU` 队列、`24` 核，且训练阶段必须观测到真实 GPU 占用

画图和 notebook 请单独使用分析环境，不要把 `seaborn`、`jupyterlab` 之类的包直接塞进训练环境里。

## 4. 所有后端都先做环境检查

进入对应工程目录后，先运行：

```bash
python scripts/check_environment.py --config configs/base.yaml --strict
python scripts/check_environment.py --config configs/base.yaml --strict --test-mlatom-xtb
```

如果 `ts_seed.xyz` 还没生成，脚本会临时从 `geometries/ture_seed/TS.log` 解析一个检查用的 XYZ，不需要先手工补文件。

## 5. 各后端推荐的一键多轮入口

### 5.1 ANI

```bash
cd minimal_adl_ethene_butadiene
python scripts/active_learning_loop.py \
  --config configs/base.yaml \
  --submit-mode-labels pbs \
  --submit-mode-train pbs \
  --submit-mode-md pbs
```

### 5.2 MACE

```bash
cd minimal_adl_ethene_butadiene_mace
python scripts/active_learning_loop.py \
  --config configs/base.yaml \
  --submit-mode-labels pbs \
  --submit-mode-train pbs \
  --submit-mode-md pbs
```

### 5.3 KREG

```bash
cd minimal_adl_ethene_butadiene_KREG
python scripts/active_learning_loop.py \
  --config configs/base.yaml \
  --submit-mode-labels pbs \
  --submit-mode-train pbs \
  --submit-mode-md pbs
```

这个入口会先跑通第一轮，再自动继续后续轮次，直到收敛或达到 `max_rounds`。
如果某一轮里只有少量样本缺 label，总控只会重提这些缺失样本，不会把整轮 selected manifest 再整批重提。

## 6. 第一轮调试入口

如果你只想调试第一轮，或者当前只想定位某个单轮问题，再用：

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --submit-mode-labels pbs \
  --submit-mode-train pbs \
  --submit-mode-md pbs
```

如果你不想一开始就上完整一轮，可以按下面的顺序做缩小版链路测试：

```bash
python scripts/sample_ts_initial_conditions.py --config configs/base.yaml --round-index 0 --num-initial-conditions 10
python scripts/run_xtb_labels.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json --submit-mode pbs
python scripts/run_target_labels.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json --submit-mode pbs
python scripts/update_cumulative_manifest.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json
python scripts/build_delta_dataset.py --config configs/base.yaml --manifest data/processed/cumulative_labeled_manifest.json
python scripts/train_main_model.py --config configs/base.yaml --submit-mode pbs
python scripts/train_aux_model.py --config configs/base.yaml --submit-mode pbs
python scripts/run_md_sampling.py --config configs/base.yaml --round-index 1 --submit-mode pbs --num-initial-conditions 4 --maximum-propagation-time 10 --time-step 0.5 --save-interval-steps 2

```
