# 训练总流程（新手详细版）

这份文档给第一次接触本仓库的同学使用，目标是：

- 先把环境配对
- 再把流程跑通
- 最后能读懂结果和分析 notebook

## 0. 仓库结构先看懂

本仓库有三个并列工程：

- ANI：`minimal_adl_ethene_butadiene`
- MACE：`minimal_adl_ethene_butadiene_mace`
- NequIP（预留）：`minimal_adl_ethene_butadiene_nequip`

`docs/` 只保留三个目录：

- `docs/ani/`
- `docs/mace/`
- `docs/nequip/`

每个目录都有 4 份核心文档：

- `环境配置.md`
- `训练流程.md`
- `结果说明.md`
- `数据分析.ipynb`

## 1. 环境和工程一一对应（非常重要）

- ANI 工程必须用：`ADL_env`
- MACE 工程必须用：`ADL_MACE`
- NequIP 工程必须用：`ADL_NequIP`

不要混用环境，否则容易出现“能 import 但训练阶段失败”的问题。

## 2. 先跑门禁检查，再开始训练

进入对应工程目录后，固定先执行：

```bash
python scripts/check_environment.py --config configs/base.yaml --strict
python scripts/check_environment.py --config configs/base.yaml --strict --test-mlatom-xtb
```

在 GPU 节点建议再加：

```bash
python scripts/check_environment.py --config configs/base.yaml --strict --expect-gpu
```

这一步通过后再训练，能省很多排错时间。

## 4. 完整第一轮训练（ANI / MACE）

ANI 或 MACE 跑完整流程（PBS）：

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --submit-mode-labels pbs \
  --submit-mode-train pbs \
  --submit-mode-uq pbs
```

主流程阶段通常包括：

1. `check_environment`
2. `sample_initial_geometries`
3. `initial_selection`
4. `run_xtb_labels`
5. `run_target_labels`
6. `build_delta_dataset`
7. `train_main_model`
8. `train_aux_model`
9. `export_training_diagnostics`
10. `evaluate_uncertainty`
11. `select_round_001`

## 5. 断点续跑（很常用）

从训练阶段继续：

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --from-stage train_main_model \
  --submit-mode-train pbs \
  --submit-mode-uq pbs
```

强制重跑：

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --from-stage train_main_model \
  --force \
  --submit-mode-train pbs \
  --submit-mode-uq pbs
```

## 6. 跑完后去哪里看结果

每个工程都看自己的目录，不要串目录：

- `models/`：模型与训练产物
- `results/`：UQ、选点、流程汇总

详细验收项请看：

- `docs/ani/结果说明.md`
- `docs/mace/结果说明.md`
- `docs/nequip/结果说明.md`

## 7. 数据分析 notebook 怎么用

三个模型都各有 notebook：

- `docs/ani/数据分析.ipynb`
- `docs/mace/数据分析.ipynb`
- `docs/nequip/数据分析.ipynb`

使用方式：

1. 先确保已经有对应工程的 `models/` 和 `results/` 产物
2. 在支持 notebook 的环境打开对应 `数据分析.ipynb`
3. 从头顺序运行，若路径识别错误，按 notebook 顶部配置提示调整项目根目录

## 8. 新手最容易踩的坑

- 混用 conda 环境（最常见）
- 没先跑 `check_environment --strict`
- 在 CPU 节点要求 GPU 检查
- `xtb` / `g16` 没在 PATH
- 跑的是 A 工程，分析时却打开 B 工程 notebook

## 9. 按模型查文档入口

- NequIP：`docs/nequip/环境配置.md` → `训练流程.md` → `结果说明.md` → `数据分析.ipynb`
