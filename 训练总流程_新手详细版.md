# 训练总流程（新手详细版）

这份文档给第一次接触本仓库的同学使用，目标是：

- 先把环境配对
- 再把流程跑通
- 最后能读懂结果和分析 notebook

## 0. 仓库结构先看懂

本仓库有三个并列工程：

- ANI：`minimal_adl_ethene_butadiene`
- MACE：`minimal_adl_ethene_butadiene_mace`
- NequIP（预留）：`minimal_adl_ethene_butadiene_nequip`

每个模型都应优先看自己的 `docs/<model>/` 文档，不要把一个模型的阶段名套到另一个模型上。

## 1. 环境和工程一一对应（非常重要）

- ANI 工程必须用：`ADL_env`
- MACE 工程必须用：`ADL_MACE`
- NequIP 工程必须用：`ADL_NequIP`

不要混用环境，否则容易出现“能 import 但训练阶段失败”的问题。

## 2. 先跑门禁检查，再开始训练

进入对应工程目录后，固定先执行：

```bash
python scripts/check_environment.py --config configs/base.yaml --strict
python scripts/check_environment.py --config configs/base.yaml --strict --test-mlatom-xtb
```

在 GPU 节点建议再加：

```bash
python scripts/check_environment.py --config configs/base.yaml --strict --expect-gpu
```

## 3. 最小联通测试

### 3.1 ANI

ANI 现在已经切到 `TS seed + 双向 MD` 流程，推荐按下面这条最小链路验证：

```bash
conda activate ADL_env
cd /share/home/Chenlehui/work/test_ADL/minimal_adl_ethene_butadiene

python scripts/prepare_ts_seed.py --config configs/base.yaml
python scripts/sample_ts_initial_conditions.py --config configs/base.yaml --round-index 0 --num-initial-conditions 10
python scripts/run_xtb_labels.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json --submit-mode local
python scripts/run_target_labels.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json --submit-mode local
python scripts/update_cumulative_manifest.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json
python scripts/build_delta_dataset.py --config configs/base.yaml --manifest data/processed/cumulative_labeled_manifest.json
python scripts/train_main_model.py --config configs/base.yaml --submit-mode local
python scripts/train_aux_model.py --config configs/base.yaml --submit-mode local
python scripts/run_md_sampling.py --config configs/base.yaml --round-index 1 --submit-mode local --num-initial-conditions 4 --maximum-propagation-time 10 --save-interval-steps 2
python scripts/select_md_frames.py --config configs/base.yaml --round-index 1 --max-new-points 5
```

### 3.2 MACE

MACE 仍按自己的 `docs/mace/训练流程.md` 执行，不要直接套 ANI 的阶段名。

### 3.3 NequIP

NequIP 当前按自己的 `docs/nequip/训练流程.md` 执行。

## 4. ANI 完整第一轮训练

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --submit-mode-labels pbs \
  --submit-mode-train pbs \
  --submit-mode-md pbs
```

ANI 这条 pipeline 的阶段顺序固定为：

1. `check_environment`
2. `prepare_ts_seed`
3. `sample_round_000_initial_conditions`
4. `labels_round_000`
5. `build_delta_dataset`
6. `train_main_model`
7. `train_aux_model`
8. `export_training_diagnostics`
9. `run_md_sampling_round_001`
10. `select_round_001_frames`

## 5. 断点续跑

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --from-stage train_main_model \
  --submit-mode-train pbs \
  --submit-mode-md pbs
```

强制重跑：

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --from-stage train_main_model \
  --force \
  --submit-mode-train pbs \
  --submit-mode-md pbs
```

## 6. 跑完后去哪里看结果

每个工程都看自己的目录，不要串目录：

- `models/`：模型与训练产物
- `results/`：轮次结果与流程汇总
- `data/processed/`：累计标注 manifest 与 delta 数据集

## 7. 数据分析 notebook 怎么用

- ANI notebook：`docs/ani/数据分析.ipynb`
- MACE notebook：`docs/mace/数据分析.ipynb`
- NequIP notebook：`docs/nequip/数据分析.ipynb`

ANI notebook 现在默认分析 TS seed、累计标注数据、MD 帧 UQ 和轮次选帧结果。

## 8. 新手最容易踩的坑

- 混用 conda 环境
- 没先跑 `check_environment --strict`
- 还在按旧的静态 `400` 点池思路理解 ANI 流程
- `xtb` / `g16` 没在 PATH
- 训练看的是一个工程，分析 notebook 打开的却是另一个工程
