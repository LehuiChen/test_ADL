# ADL_MACE CPU-default pinned pip requirements reference

# Important:
# - this file documents the exact pip-side versions used by the CPU-first MACE workflow
# - do NOT blindly run `pip install -r requirements.txt` in a fresh env
# - use environment.cpu.yml to create the conda base env first
# - then use scripts/install_mace_deps_stepwise.sh so that mlatom / mace-torch can be installed with --no-deps where required
#
# Why this matters:
# - mlatom should be installed with --no-deps to avoid pulling an incompatible torch stack
# - current cluster GPU driver is too old for the tested torch 2.6 CUDA route
# - numpy should stay on the conda side at 1.26.x to avoid matscipy ABI issues
# - xtb / g16 are system programs and must come from the cluster environment, not pip
