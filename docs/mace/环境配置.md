# MACE 环境配置（CPU 默认版）

对应工程：`minimal_adl_ethene_butadiene_mace`  
对应 conda：`ADL_MACE`  
对应模型：`MACE`

## 1. 当前机房的推荐结论

基于这次服务器实测，`minimal_adl_ethene_butadiene_mace` 目前推荐按 **CPU 节点** 运行。

原因很直接：

- 机房当前这台 3080 节点驱动较老（日志里是 `Driver Version 470.94`，`CUDA Version 11.4`）
- 当前能和 `MLatom + MACE` 较顺利配合的 PyTorch 路线需要 `torch 2.x`
- 在现有节点驱动下，PyTorch CUDA 侧无法稳定启用
- 因此本项目默认配置已切到：
  - `training.device: cpu`
  - `cluster.resources_by_method.training.queue: default`
  - `cluster.resources_by_method.uncertainty.queue: default`
  - `training/uncertainty` 默认都按 `default` 队列、`16` 核提交

重点结论：

- 现在这条 MACE 教学线，优先目标是 **CPU 跑通第一轮流程**
- 不再默认提交到 GPU 队列
- 如果以后机房换了更新驱动的 GPU 节点，再单独恢复 GPU 路线

## 2. 仓库里现成可用的环境文件

当前仓库已经提供三种入口：

- `minimal_adl_ethene_butadiene_mace/environment.cpu.yml`
- `minimal_adl_ethene_butadiene_mace/requirements.txt`
- `minimal_adl_ethene_butadiene_mace/scripts/install_mace_deps_stepwise.sh`

推荐优先级：

1. 先用 `environment.cpu.yml` 创建 conda 基础环境
2. 再运行 `scripts/install_mace_deps_stepwise.sh` 安装 pip 侧精确依赖
3. `requirements.txt` 作为 pip 版本清单和排错参考，不建议直接单独 `pip install -r`

这样安排的原因是：

- `mlatom` 和 `mace-torch` 在本项目中需要按 `--no-deps` 路线控制安装
- `environment.yml` 无法优雅表达这种“单包禁用依赖”的策略
- 所以真正的权威复现路径是：`environment.cpu.yml + install_mace_deps_stepwise.sh`

## 3. 当前建议的版本组合

这套组合优先追求“在当前集群上可复现、默认 CPU 跑通”：

- `Python 3.10`
- `torch 2.6.0`
- `mace-torch 0.3.11`
- `e3nn 0.4.4`
- `ase 3.28.0`
- `matscipy`（建议 `conda-forge` 安装）
- `mlatom 3.22.0`（必须 `--no-deps` 安装）
- `pyh5md 1.2.0`

说明：

- `ADL_MACE` 不能沿用 ANI 的 `torch 1.12.0`
- `torch 1.12` 会缺少 `torch.compiler`，不适合当前 MACE 路线
- `mace-torch 0.3.15` 在本次实测里和 `MLatom` 接口出现过不匹配迹象
- 因此这里把文档和脚本统一收紧到 `mace-torch 0.3.11`

## 8. PyTorch 2.6 兼容说明

PyTorch 2.6 默认把 `torch.load(..., weights_only)` 改成了更严格的行为，会影响 `e3nn 0.4.4` 加载 `constants.pt`。

本项目已经做了两层兼容：

- `src/minimal_adl/mlatom_bridge.py`
- `configs/base.yaml`

手动验证也可以这样做：

```bash
export TORCH_FORCE_NO_WEIGHTS_ONLY_LOAD=1
python -c "from e3nn import o3; print('e3nn load ok')"
```

## 9. 常见坑与处理

### 9.1 `Network is unreachable`

统一用国内镜像：

```bash
python -m pip install -i https://pypi.tuna.tsinghua.edu.cn/simple --retries 2 --timeout 20 <package>
```

### 9.2 `matscipy` pip 编译失败

老机房编译器常见报错，直接改用：

```bash
conda install -y -c conda-forge matscipy
```

### 9.3 `numpy.core.multiarray failed to import`

通常是 `matscipy` 和 `numpy` ABI 不匹配。先固定：

```bash
conda install -y -c conda-forge "numpy=1.26.*" matscipy --force-reinstall
```

### 9.4 `module 'torch' has no attribute 'compiler'`

说明你误用了 ANI 的 `torch 1.12`。  
MACE 环境请切回 `torch 2.6.0`。

### 9.5 `Weights only load failed`

这是 PyTorch 2.6 和旧版 `e3nn` 的兼容问题。  
先确认仓库里的 `mlatom_bridge.py` 和 `base.yaml` 已更新到最新版本。

### 9.6 `No CUDA device available!`

在当前机房节点上，这通常不是脚本写错，而是：

- 节点驱动较老
- 当前 PyTorch CUDA 后端起不来

本项目现在已经默认改成 CPU 路线，遇到这类问题时不要再优先怀疑脚本。

### 9.7 `config_from_atoms_list() missing 1 required positional argument: 'key_specification'`

这通常意味着 `MLatom` 和 `mace-torch` 版本接口不匹配。  
当前仓库默认建议使用：

```bash
mace-torch==0.3.11
```
