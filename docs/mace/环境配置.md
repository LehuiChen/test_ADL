# MACE 环境配置

对应工程：`minimal_adl_ethene_butadiene_mace`

## 1. 推荐环境路线

当前机房默认推荐 **CPU + PBS** 路线。

原因：

- 当前 MACE + MLatom 组合已经按 CPU 流程稳定适配。
- 现有 GPU 节点驱动较老时，容易出现 `torch.cuda.is_available() == False`。
- 当前主线是长时间训练 + 双向 MD，更需要稳定性。

## 2. 创建环境

```bash
cd /share/home/Chenlehui/work/test_ADL/minimal_adl_ethene_butadiene_mace
conda env remove -n ADL_MACE -y || true
conda env create -f environment.cpu.yml
conda activate ADL_MACE
bash scripts/install_mace_deps_stepwise.sh
```

## 3. 关键版本约束

当前项目默认依赖组合：

- `torch == 2.6.0`
- `mace-torch == 0.3.11`
- `e3nn == 0.4.4`
- `numpy == 1.26.x`
- 默认训练设备：`cpu`

说明：

- `mace-torch 0.3.15` 与当前 MLatom 接口不兼容，因此默认固定 `0.3.11`。
- `numpy 2.x` 与 `matscipy` 容易出现 ABI 不匹配，因此固定到 `1.26.x`。

## 4. TS / xTB / Gaussian 前置

默认 TS 输入文件在：

- `geometries/ts_seed/TS.log`

标签阶段需要保证：

- `xtb` 在 `PATH` 中
- `g16` 与 `Gau_Mlatom.py` 可用

推荐进入服务器后先执行：

```bash
source /share/apps/gaussian/g16-env.sh
source /share/env/ips2018u1.env
source ~/.bashrc
conda activate ADL_MACE
export PATH=/share/apps/gaussian/g16:/share/pubbin:/share/home/Chenlehui/bin:/share/apps/xtb-6.7.1/xtb-dist/bin:$PATH
export dftd4bin=/share/apps/dftd4-3.5.0/bin/dftd4
```

## 5. 门禁检查

```bash
python scripts/check_environment.py --config configs/base.yaml --strict
python scripts/check_environment.py --config configs/base.yaml --strict --test-mlatom-xtb
```

推荐关注：

- `mace_backend.ok == true`
- `torch.cuda_available == false` 且 `expected_gpu == false`
- `mlatom_xtb_single_point.ok == true`

这条主线默认就是 CPU，不要求 CUDA 可用。

## 6. 默认资源配置

当前 `configs/base.yaml` 默认：

- `queue: default`
- `ppn: 16`
- `walltime: 360:00:00`

这套长 walltime 是为了避免 CPU 训练和后续双向 MD 在 PBS 中途被切掉。
