# MACE 环境配置（详细版）

对应工程：`minimal_adl_ethene_butadiene_mace`  
对应 conda：`ADL_MACE`  
对应模型：`MACE`

## 1. 版本策略（先稳后快）

MACE 的后端依赖对 `torch` 版本更敏感，不同机房差异较大，建议策略：

- 固定 `Python 3.10`
- 先装与机房驱动兼容的 `torch`
- 再安装 `mace-torch`（或 `mace`）
- 用 `check_environment.py --strict` 做最终判定

如果你机房是老驱动（如 470 系列），可先尝试与 ANI 同步的 `torch 1.12 + cu11.3`；
若 `mace-torch` 与其不兼容，再切到机房允许的更高 torch 版本。

## 6. 常见坑与处理

### 6.1 `关键环境检查未通过: mace_backend`

```bash
conda activate ADL_MACE
python -m pip install mace-torch
```

### 6.2 `torch` 可用但 MACE 训练报后端错误

大概率是 `torch` 与 `mace-torch` 组合不匹配，建议在新环境重新组合安装。

### 6.3 `xtb` / `g16` 找不到

不是 Python 问题，是系统 PATH 没加载。优先检查 `which xtb`、`which g16`。

### 6.4 `--expect-gpu` 不通过

先确认 GPU 节点，再检查 `nvidia-smi` 与 torch cuda 版本。
