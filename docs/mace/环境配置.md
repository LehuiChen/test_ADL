# MACE 环境配置（实测版）

对应工程：`minimal_adl_ethene_butadiene_mace`  
对应 conda：`ADL_MACE`  
对应模型：`MACE`

## 1. 本次实测通过的版本组合

这套组合来自本次服务器排错记录，优先以“能跑通 MLatom + MACE + PBS”为准：

- `Python 3.10`
- `torch 2.6.0+cu124`
- `mace-torch 0.3.15`
- `e3nn 0.4.4`
- `ase 3.28.0`
- `matscipy`（建议 `conda-forge` 安装，不走 pip 源码编译）
- `mlatom 3.22.0`（必须 `--no-deps` 安装）
- `pyh5md 1.2.0`

重点结论：

- `ADL_MACE` 不能沿用 ANI 的 `torch 1.12.0`
- 当前 MACE 路线必须用 `torch 2.x`
- `mace-torch 0.3.15` 要求 `e3nn==0.4.4`
- PyTorch 2.6 需要兼容 `weights_only` 变更，本项目已在代码和 PBS 环境里固化处理

## 7. PyTorch 2.6 兼容说明

PyTorch 2.6 默认把 `torch.load(..., weights_only)` 改成了更严格的行为，会影响 `e3nn 0.4.4` 加载 `constants.pt`。

本项目已经做了两层兼容：

- [mlatom_bridge.py](../../minimal_adl_ethene_butadiene_mace/src/minimal_adl/mlatom_bridge.py)
- [base.yaml](../../minimal_adl_ethene_butadiene_mace/configs/base.yaml)

如果你想在当前 shell 手动验证，也可以临时执行：

```bash
export TORCH_FORCE_NO_WEIGHTS_ONLY_LOAD=1
python -c "from e3nn import o3; print('e3nn load ok')"
```

## 8. 常见坑与处理

### 8.1 `Network is unreachable`

统一用国内镜像，不要盲等：

```bash
python -m pip install -i https://pypi.tuna.tsinghua.edu.cn/simple --retries 2 --timeout 20 <package>
```

### 8.2 `h5py` / `pyh5md` 编译失败

优先让 `h5py` 走 conda，不要在没有系统 `libhdf5.so` 的节点硬编。

### 8.3 `matscipy` pip 编译失败

老机房编译器常见报错，直接改用：

```bash
conda install -y -c conda-forge matscipy
```

### 8.4 `module 'torch' has no attribute 'compiler'`

说明你误用了 ANI 的 `torch 1.12`。  
MACE 环境请切回 `torch 2.6.0`。

### 8.5 `Weights only load failed`

这是 PyTorch 2.6 和旧版 `e3nn` 的兼容问题。  
先确认仓库里的 `mlatom_bridge.py` 和 `configs/base.yaml` 已更新到最新版本。

### 8.6 `uncertainty_latest.json` 不存在

通常不是选点脚本的问题，而是训练阶段先失败了。  
先检查：

```bash
cat models/train_main_status.json
cat models/train_aux_status.json
cat results/uncertainty_status.json
```
