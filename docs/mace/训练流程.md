## 2. 第一轮主线推荐跑法（CPU / PBS）

不要一开始就直接跑 `select_round_001`。  
第一轮正确顺序是：

1. `initial_selection -> build_delta_dataset`
2. `train_main_model -> evaluate_uncertainty`
3. 最后单独跑 `select_round_001`

### 2.1 先补齐第一轮数据集

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --from-stage initial_selection \
  --to-stage build_delta_dataset \
  --submit-mode-labels pbs
```

验收：

```bash
python - <<'PY'
import json
with open('data/processed/delta_dataset_metadata.json', 'r', encoding='utf-8') as f:
    d = json.load(f)
print('delta num_samples =', d.get('num_samples'))
PY
```

### 2.2 再跑训练与不确定性评估

当前默认配置已经是 CPU，所以常规跑法不需要再额外加 `--device cpu`：

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --from-stage train_main_model \
  --to-stage evaluate_uncertainty \
  --submit-mode-train pbs \
  --submit-mode-uq pbs
```

如果前面失败过，重跑时加 `--force`：

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --from-stage train_main_model \
  --to-stage evaluate_uncertainty \
  --submit-mode-train pbs \
  --submit-mode-uq pbs \
  --force
```

如果你想显式写清 CPU，也可以：

```bash
python scripts/train_main_model.py --config configs/base.yaml --submit-mode pbs --device cpu
python scripts/train_aux_model.py --config configs/base.yaml --submit-mode pbs --device cpu
python scripts/evaluate_uncertainty.py --config configs/base.yaml --manifest data/raw/geometry_pool_manifest.json --submit-mode pbs --device cpu --output results/uncertainty_latest.json
```

验收：

```bash
cat models/train_main_status.json
cat models/train_aux_status.json
cat results/uncertainty_status.json
ls -l models/training_state.json results/uncertainty_latest.json
```

注意：

- `status.json` 里如果显示 `"device": "cpu"`，就说明训练确实在按 CPU 跑
- 即使 PBS 队列名字写的是某个节点类型，也要以 `job.pbs` 里的 `--device` 和 `status.json` 为准

### 2.3 最后再做下一轮选点

## 3. 常见续跑方式

只重跑训练与 UQ：

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --from-stage train_main_model \
  --to-stage evaluate_uncertainty \
  --submit-mode-train pbs \
  --submit-mode-uq pbs \
  --force
```

只重跑 UQ：

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --from-stage evaluate_uncertainty \
  --to-stage evaluate_uncertainty \
  --submit-mode-uq pbs \
  --force
```

如果想直接本地验证 UQ：

```bash
python scripts/evaluate_uncertainty.py \
  --config configs/base.yaml \
  --manifest data/raw/geometry_pool_manifest.json \
  --submit-mode local \
  --device cpu \
  --output results/uncertainty_latest.json
```

## 4. 为什么当前默认不用 GPU

这次服务器实测已经说明：

- 当前 3080 节点驱动较老
- 即便 `nvidia-smi` 能看到 GPU，也不代表当前 PyTorch CUDA 可用
- 因此当前仓库默认已经切到 CPU 路线，避免重复踩坑

如果以后机房提供更新驱动的 GPU 节点，再单独恢复 GPU 配置更稳。
