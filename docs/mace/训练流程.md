# MACE 训练流程

## 1. 当前主线

MACE 现在与 ANI 对齐，主线是：

`TS.log -> prepare_ts_seed -> sample_ts_initial_conditions -> round 0 labels -> build_delta_dataset -> MACE train -> bidirectional MD -> stop on first uncertain point -> select_md_frames -> next round labels`

区别只在后端和资源：MACE 当前固定 CPU-only。

## 2. 推荐的一键多轮入口

```bash
python scripts/active_learning_loop.py \
  --config configs/base.yaml \
  --submit-mode-labels pbs \
  --submit-mode-train pbs \
  --submit-mode-md pbs
```

这个入口会先跑通第一轮，再自动继续后续轮次；MACE 当前固定 CPU-only，如果需要显式指定也可以加 `--device cpu`。

实验总控过程会写到：

- `results/active_learning_experiment_summary.json`
- `results/active_learning_round_history.json`

其中 `active_learning_experiment_summary.json` 会汇总：

- 总共跑到第几轮
- 每轮加入了多少新点
- 当前累计数据集大小
- 最终停止原因（收敛 / 达到最大轮数 / 中途失败）
- 如果某一轮里个别 baseline / target 标签缺失，总控会自动重提并重试，而不是立刻要求你手动补
- 自动重试只会补交当前仍缺 label 的样本，不会把整轮 selected manifest 整批重复提交

```bash
python scripts/select_md_frames.py --config configs/base.yaml --round-index 1 --max-new-points 5
```

## 5. 关于 MD 这一步

- round 0 初始条件会全部标注
- 后续 MACE MD 使用当前 MACE delta 势传播
- 每条轨迹一旦第一次超过不确定度阈值，就在那个点停下
- 只有这些返回点才进入下一轮筛选与标注
- 默认不再把新点硬截成 100 个
- 如果你只想临时限制预算，再显式传 `--max-new-points`

## 6. 当前默认参数

在 `configs/base.yaml` 里，默认设置是：

- `initial_conditions_initial: 200`
- `initial_conditions_per_round: 100`
- `trajectory_mode: bidirectional`
- `maximum_propagation_time: 150`
- `time_step: 0.5`
- `save_interval_steps: 5`
- `ensemble: NVE`
- `max_new_points_per_round: null`（默认不设每轮新点上限）
- 所有阶段默认走 `N4` 队列、`24` 核
- baseline worker 默认 `4` 个，target worker 默认 `4` 个
