## 3. 结果怎么判定是对的

- `ts_seed_summary.json` 里应看到 `16` 个原子、`Charge=0`、`Multiplicity=1`、`1` 个虚频
- `cumulative_labeled_manifest.json` 的 `sample_id` 应与 label 目录完全一致
- `delta_dataset_metadata.json` 的 `num_samples` 应与累计 manifest 一致
- `round_001_md_frame_manifest.json` 应包含“每条轨迹首个不确定点”的 `sample_id / trajectory_id / frame_index / time_fs / uncertainty`
- `round_001_md_trajectory_summary.json` 里应能看到 `stopped_by_uncertainty`
- `round_001_selection_summary.json` 应包含 `selected_count`、`uncertain_ratio`、`converged`

## 4. 退役说明

旧的静态几何池主流程已经退役，当前验收与分析都只看 TS seed、累计标注数据、MD 返回点 manifest 和轮次选点结果。

## 5. 结果含义

`round_001_md_frame_manifest.json` 不代表“整条轨迹所有存档帧”，而代表“每条轨迹在首次超过 UQ 阈值时返回的那个点”。

所以当前主动学习逻辑是：

1. 用当前 MACE 势跑双向 MD
2. 轨迹一旦遇到首个不确定点就停下
3. 只把这个点送回去做下一轮重标注
4. 再用累计数据重建数据集并重新训练
