# ANI 训练流程

## 1. 当前主线

ANI 现在的主线是：

`TS.log -> prepare_ts_seed -> sample_ts_initial_conditions -> round 0 labels -> build_delta_dataset -> ANI train -> bidirectional MD -> stop on first uncertain point -> select_md_frames -> next round labels`

这已经不是旧的“静态 400 点几何池”流程。

## 2. 推荐的一键多轮入口

```bash
python scripts/active_learning_loop.py \
  --config configs/base.yaml \
  --submit-mode-labels pbs \
  --submit-mode-train pbs \
  --submit-mode-md pbs
```

这个入口会先跑通第一轮，然后自动继续：

1. 标注当前轮返回的不确定点
2. 更新累计数据集
3. 重建 delta dataset
4. 重训主模型和辅助模型
5. 跑下一轮 MD
6. 再选出下一轮不确定点
7. 直到收敛或者达到 `max_rounds`

## 3. 第一轮调试入口

```bash
python scripts/run_first_round_pipeline.py \
  --config configs/base.yaml \
  --submit-mode-labels pbs \
  --submit-mode-train pbs \
  --submit-mode-md pbs
```

这个入口会按顺序执行：

1. `check_environment`
2. `prepare_ts_seed`
3. `sample_round_000_initial_conditions`
4. `labels_round_000`
5. `build_delta_dataset`
6. `train_main_model`
7. `train_aux_model`
8. `export_training_diagnostics`
9. `run_md_sampling_round_001`
10. `select_round_001_frames`

这个入口仍然保留，适合只调试第一轮或者排查单轮问题。

## 4. 最小 smoke test

如果你想先做一个缩小版链路测试，可以按下面的顺序跑：

```bash
python scripts/prepare_ts_seed.py --config configs/base.yaml
python scripts/sample_ts_initial_conditions.py --config configs/base.yaml --round-index 0 --num-initial-conditions 10
python scripts/run_xtb_labels.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json --submit-mode pbs
python scripts/run_target_labels.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json --submit-mode pbs
python scripts/update_cumulative_manifest.py --config configs/base.yaml --manifest results/round_000_initial_conditions_manifest.json
python scripts/build_delta_dataset.py --config configs/base.yaml --manifest data/processed/cumulative_labeled_manifest.json
python scripts/train_main_model.py --config configs/base.yaml --submit-mode pbs
python scripts/train_aux_model.py --config configs/base.yaml --submit-mode pbs
python scripts/run_md_sampling.py --config configs/base.yaml --round-index 1 --submit-mode pbs --num-initial-conditions 4 --maximum-propagation-time 10 --time-step 0.5 --save-interval-steps 2
python scripts/select_md_frames.py --config configs/base.yaml --round-index 1 --max-new-points 5
```

## 5. 关于 MD 这一步

- round 0 的初始条件会全部做 baseline / target 标注
- 后续 MD 是用当前 ANI delta 势传播轨迹
- 每条轨迹一旦第一次超过不确定度阈值，就在那个点停下
- 只有这些返回的候选点才会进入下一轮筛选和标注
- 不是把整条轨迹上的所有点都拿去做 Gaussian
- 当前默认不再把新点硬截成 100 个
- 只要是返回的不确定点，就都会进入下一轮标注候选；如果你想临时限额，再显式传 `--max-new-points`

## 6. 当前默认参数

在 `configs/base.yaml` 里，默认设置是：

- `initial_conditions_initial: 200`
- `initial_conditions_per_round: 100`
- `trajectory_mode: bidirectional`
- `maximum_propagation_time: 150`
- `time_step: 0.5`
- `save_interval_steps: 5`
- `ensemble: NVE`
- `max_new_points_per_round: null`（默认不设每轮新点上限）
