# ANI 结果说明

## 1. 先看哪些目录

ANI 的结果只在自己的工程目录里看：

- `results/`
- `models/`
- `data/processed/`
- `labels/xtb/`
- `labels/gaussian/`

## 2. TS seed 相关结果

- `results/ts_seed_summary.json`
  记录 TS log 解析结果，例如原子数、电荷、多重度、虚频数量、首个虚频。
- `geometries/ture_seed/ts_seed.json`
  标准化后的 MLatom JSON seed。
- `geometries/ture_seed/ts_seed.xyz`
  标准化后的 XYZ seed。

## 3. round 0 结果

- `results/round_000_initial_conditions_manifest.json`
  round 0 的 TS 初始条件清单。
- `results/round_000_labels_status.json`
  round 0 baseline / target 标注的汇总状态。
- `data/processed/cumulative_labeled_manifest.json`
  当前累计已标注样本。
- `data/processed/delta_dataset_metadata.json`
  delta 数据集统计信息。

## 4. 训练结果

- `models/train_main_status.json`
- `models/train_aux_status.json`
- `models/training_state.json`
- `models/training_diagnostics.json`

如果 `train_main_status.json` 和 `train_aux_status.json` 里的 `success` 都是 `true`，说明这轮模型训练成功。

## 5. MD 与选点结果

- `results/round_001_md_sampling_status.json`
- `results/round_001_md_frame_manifest.json`
- `results/round_001_selection_summary.json`
- `results/round_001_selected_manifest.json`
- `results/active_learning_round_history.json`

重点字段：

- `sampling_logic = stop_on_first_uncertain_point`
- `selection_basis = first_uncertain_md_point`
- `save_interval_steps`
- `dumped_num_steps`
- `selected_count`
- `converged`

`converged = false` 不是报错，只表示当前轮次之后仍然找到了足够多的新不确定点，还没有收敛。

## 6. 导出轨迹怎么理解

现在导出的轨迹文件会真正使用 `save_interval_steps` 做抽稀：

- 第一帧会保留
- 每隔 `save_interval_steps` 步导出一帧
- 最后一帧 / 停止帧会保留

这只影响导出的轨迹密度，不改变“首个不确定点即停止”的选点逻辑。
