# ANI 环境配置

对应工程：`minimal_adl_ethene_butadiene`  
对应 conda：`ADL_env`  
对应模型：`ANI`

## 1. 本环境负责什么

`ADL_env` 用于 ANI 工程的完整主流程：

- 数据准备与标注阶段
- `train_main_model` / `train_aux_model`
- UQ 与选点
- `check_environment.py` 训练前门禁检查

## 2. 依赖清单（按实际检查脚本）

`scripts/check_environment.py --strict` 会重点检查：

- Python 模块：`yaml`、`mlatom`、`pyh5md`、`joblib`、`sklearn`、`torch`、`torchani`
- 系统命令：`xtb`
- 当 `target.program=Gaussian` 时还会检查：`g16`

建议同时准备（用于分析/可视化）：

- `pandas`、`matplotlib`、`seaborn`

## 3. 创建与安装（推荐顺序）

### 3.1 新建环境

```bash
conda create -n ADL_env python=3.10 -y
conda activate ADL_env
```

### 3.2 安装项目基础依赖

```bash
cd /share/home/Chenlehui/work/test_ADL/minimal_adl_ethene_butadiene
python -m pip install -r requirements.txt
```

### 3.3 安装 ANI 相关依赖

```bash
# 先安装与你机房驱动/CUDA兼容的 torch
# 再安装 torchani
python -m pip install torchani
```

说明：ANI 下 torch / CUDA 版本兼容性很关键，建议沿用机房已验证组合。

## 4. 系统程序检查（非 pip 包）

以下程序通常由机房环境提供：

- `xtb`
- `g16`（如果启用 Gaussian target）

检查命令：

```bash
which xtb
xtb --version
which g16
```

## 5. 训练前门禁（必须先跑）

进入 ANI 工程目录后执行：

```bash
conda activate ADL_env
cd /share/home/Chenlehui/work/test_ADL/minimal_adl_ethene_butadiene
python scripts/check_environment.py --config configs/base.yaml --strict
python scripts/check_environment.py --config configs/base.yaml --strict --test-mlatom-xtb
```

GPU 节点建议附加：

```bash
python scripts/check_environment.py --config configs/base.yaml --strict --expect-gpu
```

建议留档：

```bash
python scripts/check_environment.py --config configs/base.yaml --strict --json-output results/check_environment_latest.json
```

## 6. 常见报错与处理

### 6.1 `关键环境检查未通过: torchani`

处理：

```bash
conda activate ADL_env
python -m pip install torchani
```

### 6.2 `关键环境检查未通过: xtb`

原因：`xtb` 不在当前 `PATH`。  
处理：在登录脚本或 PBS `env_blocks` 中补机房 `xtb` 路径。

### 6.3 `--expect-gpu` 下 CUDA 不可见

原因：当前不在 GPU 节点，或驱动/torch-cuda 不匹配。  
处理：切到 GPU 节点重测，并确认 `nvidia-smi` 与 torch 版本匹配。

### 6.4 `关键环境检查未通过: g16`

仅在配置里使用 Gaussian target 时出现。  
处理：确认 Gaussian 环境脚本已加载，且 `g16` 可执行。

## 7. 配置对应关系（必须一致）

`configs/base.yaml` 中应保持：

- `training.ml_model_type: ANI`
- `cluster.conda_env: ADL_env`
