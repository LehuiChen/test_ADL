# ANI 环境配置（详细版）

对应工程：`minimal_adl_ethene_butadiene`  
对应 conda：`ADL_env`  
对应模型：`ANI`

## 1. 推荐版本组合（机房老驱动友好）

这套组合来自你之前已跑通过的实践路径，优先保证稳定：

- `Python 3.10`
- `PyTorch 1.12.0`
- `torchvision 0.13.0`
- `torchaudio 0.12.0`
- `cudatoolkit 11.3`
- `torchani 2.2`
- `mlatom`（安装时用 `--no-deps`）
- `pyh5md`

> 重点：`torch + cudatoolkit + torchani` 这组版本关系最关键。

## 2. 一次性创建命令（建议原样执行）

```bash
source ~/.bashrc
conda create -n ADL_env python=3.10 -y
conda activate ADL_env

conda install -y -n base conda-libmamba-solver
conda config --set solver libmamba
conda config --set channel_priority flexible

conda install -y -c conda-forge numpy scipy pyyaml matplotlib
conda install -y -c conda-forge joblib scikit-learn h5py statsmodels tqdm
python -m pip install pyh5md

conda install -y pytorch==1.12.0 torchvision==0.13.0 torchaudio==0.12.0 cudatoolkit=11.3 -c pytorch -c conda-forge -c defaults

conda install -y -c conda-forge pandas pyarrow rich typer ase zarr numcodecs fasteners huggingface_hub httpx ninja

python -m pip install --upgrade pip setuptools wheel
python -m pip install "torchani==2.2" --no-deps
python -m pip install mlatom --no-deps
```

## 3. 为什么 `mlatom` 要 `--no-deps`

- 直接 `pip install mlatom` 可能把 `torch` 自动升级到 2.x
- 一旦 `torch` 漂移，ANI 在老驱动节点会不稳定甚至直接失败

所以固定原则：

```bash
python -m pip install mlatom --no-deps
```

## 4. 系统程序（不是 pip 包）

ANI 训练流程仍依赖机房程序：

- `xtb`
- `g16`（当 target 配置为 Gaussian）

建议在 shell 里先加载：

```bash
source /share/apps/gaussian/g16-env.sh
source /share/env/ips2018u1.env
source ~/.bashrc
conda activate ADL_env
export PATH=/share/apps/gaussian/g16:/share/pubbin:/share/home/Chenlehui/bin:/share/apps/xtb-6.7.1/xtb-dist/bin:$PATH
export dftd4bin=/share/apps/dftd4-3.5.0/bin/dftd4
```

检查：

```bash
which xtb
xtb --version
which g16
```

## 5. 训练前门禁（必须）

```bash
conda activate ADL_env
cd /share/home/Chenlehui/work/test_ADL/minimal_adl_ethene_butadiene
python scripts/check_environment.py --config configs/base.yaml --strict
python scripts/check_environment.py --config configs/base.yaml --strict --test-mlatom-xtb
```

这里的 `--test-mlatom-xtb` 现在优先读取 `geometries/ture_seed/ts_seed.xyz`；如果它还没生成，就会临时从 `geometries/ture_seed/TS.log` 解析 TS seed 再做单点测试。

GPU 节点再加：

```bash
python scripts/check_environment.py --config configs/base.yaml --strict --expect-gpu
```

建议保存报告：

```bash
python scripts/check_environment.py --config configs/base.yaml --strict --json-output results/check_environment_latest.json
```

- `torch = 1.12.0`

## 7. 常见坑

- `torchani` 没装好
- `xtb` 不在 `PATH`
- `g16` 没加载成功
- GPU 节点上 `torch.cuda.is_available()` 仍为 `False`
- `torch` 被别的 `pip install` 偷偷升级到 2.x

## 8. 配置文件必须对应

`configs/base.yaml` 中：

- `training.ml_model_type: ANI`
- `cluster.conda_env: ADL_env`
- `paths.ts_frequency_source: geometries/ture_seed/TS.log`
- `paths.cumulative_labeled_manifest: data/processed/cumulative_labeled_manifest.json`
- `sampling.initial_conditions_initial: 200`
- `sampling.initial_conditions_per_round: 100`
- `sampling.md.trajectory_mode: bidirectional`
