# NequIP 环境配置

对应工程：`minimal_adl_ethene_butadiene_nequip`  
对应 conda：`ADL_NequIP`  
对应模型：`NequIP`（当前为预留占位）

## 1. 本环境负责什么

`ADL_NequIP` 当前主要用于：

- 前置数据阶段（到 `build_delta_dataset`）
- 训练前门禁检查
- 为后续 NequIP 后端实装提前准备依赖

注意：当前仓库中 NequIP 训练分支会返回明确 `NotImplementedError`。

## 2. 依赖清单（按实际检查脚本）

`scripts/check_environment.py --strict` 会重点检查：

- Python 模块：`yaml`、`mlatom`、`pyh5md`、`joblib`、`sklearn`、`torch`
- NequIP 后端：`nequip`
- 系统命令：`xtb`
- 当 `target.program=Gaussian` 时还会检查：`g16`

建议同时准备（用于分析/可视化）：

- `pandas`、`matplotlib`、`seaborn`

## 3. 创建与安装（推荐顺序）

### 3.1 新建环境

```bash
conda create -n ADL_NequIP python=3.10 -y
conda activate ADL_NequIP
```

### 3.2 安装项目基础依赖

```bash
cd /share/home/Chenlehui/work/test_ADL/minimal_adl_ethene_butadiene_nequip
python -m pip install -r requirements.txt
```

### 3.3 安装 NequIP 依赖

```bash
python -m pip install nequip
```

## 4. 系统程序检查（非 pip 包）

以下程序通常由机房环境提供：

- `xtb`
- `g16`（如果启用 Gaussian target）

检查命令：

```bash
which xtb
xtb --version
which g16
```

## 5. 训练前门禁（必须先跑）

进入 NequIP 工程目录后执行：


## 6. 常见报错与处理

### 6.1 `关键环境检查未通过: nequip`

处理：

```bash
conda activate ADL_NequIP
python -m pip install nequip
```

### 6.2 `关键环境检查未通过: xtb`

原因：`xtb` 不在 `PATH`。  
处理：在登录脚本或 PBS `env_blocks` 中补机房 `xtb` 路径。

### 6.3 训练阶段报 `NotImplementedError`

这是当前阶段的预期行为，不是环境损坏。  
当前目标是“前置阶段可跑通 + 报错清晰可解释”。

### 6.4 `关键环境检查未通过: g16`

仅在 Gaussian target 开启时要求。  
处理：确认 Gaussian 环境脚本可用，`g16` 在 PATH 中。

## 7. 配置对应关系（必须一致）

`configs/base.yaml` 中应保持：

- `training.ml_model_type: NequIP`
- `cluster.conda_env: ADL_NequIP`
