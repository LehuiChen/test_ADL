# KREG 环境配置（详细版）
对应工程：`minimal_adl_ethene_butadiene_KREG`  
对应 conda：`ADL_KREG`  
对应模型：`KREG`

## 1. 推荐版本组合

KREG 后端由 `MLatom` 内置提供，所以这套环境的重点不是 `torchani`，而是：
- `Python 3.10`
- `mlatom`
- `numpy`
- `PyYAML`
- `matplotlib`
- `joblib`
- `scikit-learn`
- `pyh5md`

## 2. 一个最小环境思路
```bash
source ~/.bashrc
conda create -n ADL_KREG python=3.10 -y
conda activate ADL_KREG

conda install -y -c conda-forge numpy scipy pyyaml matplotlib
conda install -y -c conda-forge joblib scikit-learn h5py statsmodels tqdm
python -m pip install pyh5md
python -m pip install --upgrade pip setuptools wheel
python -m pip install mlatom --no-deps
```

## 3. 为什么 `mlatom` 仍建议 `--no-deps`

- 这样最稳，不会把机房上已经验证过的依赖链偷偷改坏。
- KREG 用的是 MLatom 内置后端，不需要额外安装 `torchani`。

## 4. 系统程序（不是 pip 包）

KREG 训练流程仍依赖机房程序：

- `xtb`
- `g16`（当 target 配置为 Gaussian）

建议在 shell 里先加载：
```bash
source /share/apps/gaussian/g16-env.sh
source /share/env/ips2018u1.env
source ~/.bashrc
conda activate ADL_KREG
export PATH=/share/apps/gaussian/g16:/share/pubbin:/share/home/Chenlehui/bin:/share/apps/xtb-6.7.1/xtb-dist/bin:$PATH
export dftd4bin=/share/apps/dftd4-3.5.0/bin/dftd4
```

检查：

```bash
which xtb
xtb --version
which g16
```

## 5. 训练前门禁（必须）
```bash
conda activate ADL_KREG
cd /share/home/Chenlehui/work/test_ADL/minimal_adl_ethene_butadiene_KREG
python scripts/check_environment.py --config configs/base.yaml --strict
python scripts/check_environment.py --config configs/base.yaml --strict --test-mlatom-xtb
```

这里的 `--test-mlatom-xtb` 现在优先读取 `geometries/ture_seed/ts_seed.xyz`；如果它还没生成，脚本会提示你先运行 `prepare_ts_seed.py`。

## 6. 常见坑
- `mlatom` 没装好
- `xtb` 不在 `PATH`
- `g16` 没加载成功
- TS seed 还没准备好就直接跑 `--test-mlatom-xtb`
- 本地 Windows 的 `python` 仍然指向 Python 2，而不是 Python 3

## 7. 配置文件必须对应

`configs/base.yaml` 中：

- `training.ml_model_type: KREG`
- `cluster.conda_env: ADL_KREG`
- `paths.ts_frequency_source: geometries/ture_seed/TS.log`
- `paths.cumulative_labeled_manifest: data/processed/cumulative_labeled_manifest.json`
- `sampling.initial_conditions_initial: 200`
- `sampling.initial_conditions_per_round: 100`
- `sampling.md.trajectory_mode: bidirectional`
