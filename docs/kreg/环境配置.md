# KREG 环境配置

这条线只负责 KREG 训练、标注和 MD。训练环境与分析环境应当分开。

## 1. 训练环境

- conda 环境名：`ADL_KREG`
- 后端：`KREG`
- 训练 / MD 设备：CPU only
- 关键外部程序：`xtb`、`g16`、`Gau_Mlatom.py`

KREG 后端由 MLatom 自带，不需要额外安装 `torchani` 或 `mace` 后端包。

## 2. 训练环境里至少要能通过的检查

```bash
python scripts/check_environment.py --config configs/base.yaml --strict
python scripts/check_environment.py --config configs/base.yaml --strict --test-mlatom-xtb
```

严格模式下，KREG 会要求：

- `yaml`
- `mlatom`
- `pyh5md`
- `joblib`
- `sklearn`
- `xtb`
- `g16`（如果 target method 使用 Gaussian）

`torch` 会被报告，但不是 KREG 的严格依赖。

## 3. 服务器启动建议

```bash
source /share/apps/gaussian/g16-env.sh
source /share/env/ips2018u1.env
source ~/.bashrc
conda activate ADL_KREG
export PATH=/share/apps/gaussian/g16:/share/pubbin:/share/home/Chenlehui/bin:/share/apps/xtb-6.7.1/xtb-dist/bin:$PATH
export dftd4bin=/share/apps/dftd4-3.5.0/bin/dftd4
```

KREG 当前固定 CPU-only。

当前配置里，KREG 的 CPU 任务 `walltime` 按服务器上限统一放宽到 `360:00:00`，包括 baseline / target / training / uncertainty / md_sampling。
当前默认资源策略是：

- 所有阶段都走 `N4` 队列
- 每个 PBS 作业默认请求 `24` 核
- baseline worker 默认 `4` 个，每个 worker 内并发 `6` 个 xTB 样本
- target worker 默认 `4` 个，每个 worker 内并发 `3` 个 Gaussian 样本

## 4. 分析环境单独准备

推荐单独准备一个 notebook / 画图环境，例如：

```bash
conda create -n ADL_analysis_kreg python=3.10 -y
conda activate ADL_analysis_kreg
python -m pip install jupyterlab pandas matplotlib seaborn pyyaml
```
